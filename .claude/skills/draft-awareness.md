# Draft Awareness

Everything in this project is a draft. Nothing is fixed.

When working on any area of clawdance:

1. **Do not treat earlier decisions as settled.** ADRs, roadmap items,
   research findings, and code may all need to change based on what we
   learn. Earlier does not mean more certain.

2. **Check cross-impacts.** When modifying any document, decision, or code,
   consider whether the change affects other parts of the project. If it
   does, update those parts too — don't leave inconsistencies.

3. **Challenge assumptions.** If something in the roadmap or docs doesn't
   match current understanding, flag it and propose changes. Don't work
   around a wrong assumption — fix it.

4. **The roadmap work items (A-E) are unordered.** We work on whatever
   makes sense. The product workflow steps (1-6) have a defined order
   but the workflow itself is a draft too.

5. **Refactoring is always valid.** If a better structure, approach, or
   architecture becomes apparent, pursue it. Sunk cost is not a reason
   to keep a bad decision.
